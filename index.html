  • HTML creates structure (frame), made up of elements such as Text, Image, Header and List
  • Element types used to describe the elements: Meta-related elements, Content division elements, Semantic elements
  • !DOCTYPE html - Declaration (not case sensitive) lets the browser know what to expect
  • head element contains the Metadata for the web application or browser
  • Metadata = data about the webpage that the browser uses to serve the page to users. Not seen by user but contains important info
  • Lang=”en” - Attribute = tells the browser the language of the webpage
  • ATTRIBUTES are always defined in the opening element, never the closing
  • title element defines the title for the webpage (required/extremely important for SEO)
  • style - element defines the CSS styles associated
  • base elements define the base URL
  • link element connects an external resource to the HTML document
  • meta element defines metadata like character set, description, keywords, author, and viewport
  • UTF-8 - allows you to use most characters and symbols on the page
  • HTTP-equiv - sets an HTTP header for content
  • Viewport - sets the default view of our page
  • script element embeds client-side scripts in the HTML document
  • body - all content shown to the user
  • h1, h2, h3, h4, h5, h6 - represent the level of heading (h1 being largest)
  • p and /p - represent a paragraph block of text, MUST be opened and closed
  • ul, ol, li - represent unordered lists and list items
  • img - contains information about images displayed on the webpage
  • br - creates a line of empty space, or line break
  • a - Anchor (creates link to the same webpage, or other webpages
  • Href - points to the URL for the link
  • section - are semantic elements that define a section in a document that contains thematic content and allows you to elegantly organize the content on your page
  • id - attribute used to identify elements and select them
  • class - works simlarly to id

CSS

  • Cascading Style Sheets (CSS)
  • There are 3 ways to style a webpage using CSS: Inline CSS, Internal CSS style sheet, and External CSS style sheet
  • Inline CSS - When a developer wants to style an element directly in the HTML file, they use inline CSS. Inline styles apply to a specific HTML tag
  • Internal CSS Style Sheet - This internal style sheet contains CSS rules for the webpage in the head section of the HTML file
  • External CSS Style Sheet - the most common way to style a webpage - allows developers to keep all their CSS rules in a separate file, which makes design changes easier
  • CSS Rule: Begins with Selector - Selector (ex; div or *). defines the element or attributes to which the rules, or declarations, will apply
  • * = a special wild card symbol used to declare that ALL the elements on the page will have the style properties margin and padding set at 0
  • , = used to combine 2 selectors into a single CSS rule, allowing declarations to affect both
  • ..followed by Declaration Block(s) (ex; color, text align, etc.) contain two components: the CSS property we want to apply and the value of the property
  • DRY code = (do not repeat yourself code) a coding principle that encourages developers to reuse or share pieces of code to reduce the number of lines of code that need to be written or maintained
  • Margin property indicates how much space we want around the outside of an element - not visible, merely adds a buffer
  • Padding property adds space around the content inside an element
  • Img - a selector that assigns property values to any image element in the markup
  • .card selector - class selector that contains width, border, shadowline properties, etc
  • Display: block; - assigns each image to start on a new line
  • ul - element that declares the font to be 20px and the padding on the left of the unordered list to be 40px
  • link element - connects to the external file
  • rel attribute - specifies the relationship between current document and linked document resource
  • href attribute - specifies the location of the external source (URL). The href attribute is using a relative path. This means the location is derived from the current location
